🔌 Installation

1. Node.js and NPM

Install the NPM package manager to manage the application's dependencies and run scripts. You need to have Node.js installed to use NPM. You can download and install Node.js from the official website https://nodejs.org/en/download


Download and install the latest version of Node.js, which includes NPM, the Node.js package manager. You can download the installer from the official website and follow the installation instructions provided for your operating system.


After installing Node.js, you can verify the installation by running the following commands in your terminal or command prompt:


node --version
v23.9.0

2. MongoDB (Database)

Install MongoDB, a NoSQL database used to store application data. You can download and install MongoDB from the official website https://www.mongodb.com/try/download/community


Download and install the latest version of MongoDB, following the installation instructions provided for your operating system. After installing MongoDB, you can start the MongoDB server by running the following command in your terminal or command prompt:


mongod

You can also use MongoDB Atlas, a cloud-based database service, to host your MongoDB database. MongoDB Atlas provides a free tier for development and testing purposes, allowing you to create a cloud-hosted MongoDB database without the need to install and manage a local database server.

https://www.mongodb.com/atlas/database

3. GhostScript & GraphicsMajick

For PDF processing (converting uploaded PDF files of question papers, answer keys and answer sheets into images), you need to install GhostScript and GraphicsMagick on your system.


For macOS:

brew install graphicsmagick
brew install ghostscript

For Linux:

sudo apt-get update
sudo apt-get install graphicsmagick
sudo apt-get install ghostscript

For Windows:

You download and install GraphicsMagick and GhostScript from their official websites:

  • GraphicsMagick
  • GhostScript

Verify Installation:

After installing GhostScript and GraphicsMagick, you can verify the installation by running the following commands in your terminal or command prompt:


gm -version
GraphicsMagick 1.3.36 2021-06-06 Q8 http://www.GraphicsMagick.org/

gs --version
9.54.0

⚙️ Backend Setup

After installing Node.js and MongoDB, you can proceed with setting up the backend of EvaluateAI. The backend is responsible for handling API requests, managing data, and performing AI-powered evaluation of answer sheets. Follow the steps below to configure the backend:


Navigate to the server directory (Projects/server) of the EvaluateAI project. This directory contains the backend codebase, i.e the Node.js application.

1. Install Dependencies

Install the required dependencies for the backend by running the following command in your terminal or command prompt:


pnpm install

2. Open the directory in your code editor

Open the backend directory in your preferred code editor. You can use Visual Studio Code, Sublime Text, Atom, or any other code editor of your choice. If you are using Visual Studio Code, you can open the directory by running the following command from the backend directory in your terminal or command prompt:


code .

3. Configure Environment Variables

Open the .env file in the backend directory and configure the environment variables based on your environment and preferences. The .env file contains settings such as the database connection URI, server port, and API keys.



Replace the following placeholders with your actual values:

  • DB_URL: MongoDB connection URI, including the database name.
  • OPENROUTER_API_KEY: OpenRouter API key for accessing the AI models.

    • EvaluateAI uses OpenRouter for accessing AI models. You can obtain an API key from https://openrouter.ai/settings/keys and use it to access the AI models for evaluating answer sheets.



  • TOKEN_SECRET: Secret key for generating JSON Web Tokens (JWT). It can be any random string.
  • STRIPE_SECRET_KEY: Secret key for integrating Stripe payments.

    • You can sign up for a free account on Stripe and obtain the publishable key for your application. https://dashboard.stripe.com/

  • RAZORPAY_KEY_ID: Key ID and secret for integrating Razorpay payments.

    • You can sign up for a free account on Razorpay and create a new key to obtain the key ID for your application. https://dashboard.razorpay.com/app/website-app-settings/api-keys

  • RAZORPAY_KEY_SECRET: Key secret for integrating Razorpay payments.
  • SMTP_HOST: SMTP host for sending email notifications (e.g., smtp.gmail.com).
  • SMTP_PORT: SMTP port for sending email notifications (e.g., 465).
  • SMTP_USER: SMTP username for sending email.
  • SMTP_PASS: SMTP password for sending email.

4. Basic Configuration

Open the server/utils/config.js file in the server directory and configure the following variables.

  • appName: Used to identify your app. It is used in Email templates etc.
  • defaultEvaluatorLimit: Default Evaluator Limit for a new user.
  • defaultEvaluationLimit: Default Evaluatoion Limit for a new user.
  • defaultClassesLimit: Default Classes Limit for a new user.
  • skipEmailVerification: Setting this to true will skip email verification for new users.
  • currency: Currency (for eg. USD, INR) used for payment gateways.
  • logoBase64: Logo base64 used for email template. You can create a base64 logo from https://www.base64-image.de/

5. Configure AI
Optional

Open the server/utils/ai.js file in the server directory and configure the following variables.

  • aiPrompt: AI prompt for evaluating answer sheets. Only change it if it's very necessary.
  • maxTokens: Maximum tokens used for AI. Increasing this will allow you to upload more questions, answer keys, and answer sheets to AI.

5. Start the Backend Server

Start the backend server by running the following command in your terminal or command prompt:


pnpm start

🖥️ Frontend Setup

After configuring the backend, you can proceed with setting up the frontend of EvaluateAI. Follow the steps below to configure the frontend:


Navigate to the web directory (Projects/web) of the EvaluateAI project. This directory contains the frontend codebase, i.e the Next.js application.

1. Install Dependencies

Install the required dependencies for the frontend by running the following command in your terminal or command prompt:


pnpm install

2. Open the directory in your code editor

Open the web directory in your preferred code editor. You can use Visual Studio Code, Sublime Text, Atom, or any other code editor of your choice. If you are using Visual Studio Code, you can open the directory by running the following command from the frontend directory in your terminal or command prompt:


code .

3. Configure Environment Variables

Open the .env.local file in the frontend directory and configure the environment variables based on your preferences.



Replace the following placeholders with your actual values:

  • NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY: Stripe Publishable Key for integrating Stripe payments. (refer backend setup)
  • NEXT_PUBLIC_RAZORPAY_KEY_ID: Razorpay Key ID for integrating Razorpay payments. (refer backend setup)
  • UPLOADTHING_TOKEN: UploadThing token.

    • EvaluateAI uses UploadThing for uploading question papers, answer keys, and answer sheets to the cloud storage service. You can sign up for a free account on UploadThing and obtain the secret key and application ID for your application. Follow the steps below to create an account and obtain the required keys:

    • Sign up for a free account on UploadThing.
    • Go to Dashboard > Create a new app.
    • Copy UPLOADTHING_TOKEN from under the SDK v7+ tab. You have to use UPLOADTHING_SECRET from Legacy tab for the Mobile App setup.

4. Basic Configuration


Open the web/src/utils/config.ts file and configure the following variables:



Replace the following placeholders with your actual values:

  • appName: Name of the application (e.g., "EvaluateAI").
  • appURL: URL of the hosted / local web app. (eg. http://localhost:3000)
  • serverURL: URL of the hosted / local backend (Node.js) app. (eg. http://localhost:8800)
  • primaryColor: Primary color theme.
  • currencySymbol: Currency symbol for payment transactions (e.g., "₹").
  • currency: Currency for payment transactions (e.g., "USD", "INR").
  • apkURL: APK URL of the mobile app for showing in the landing page.
  • onboardingSteps: For Onboarding (tutorial after welcoming user for first time).

5. Start the Frontend Server

Start the frontend server by running the following command in your terminal or command prompt:


npm run dev

📱 Mobile App Setup

EvaluateAI mobile is built using Flutter. Support both Android and iOS. Follow the steps below to set up the mobile app:

1. Install Flutter

Install Flutter, the cross-platform framework for building mobile applications, by following the installation instructions provided on the official website: https://flutter.dev/docs/get-started/install


Download and install the latest version of Flutter, including the Flutter SDK and development tools. You can follow the installation instructions provided for your operating system to set up Flutter on your local machine.

2. Configure Mobile App

Open the mobile directory (Projects/mobile) of the EvaluateAI project in your preferred code editor. This directory contains the Flutter mobile app codebase, including the Dart application and user interface components.

3. Start the Mobile App

Start the mobile app by running the following commands in your terminal:


flutter pub get
flutter run

Or you can run the app from VS Code by selecting the target device and clicking on the Run button.

Make sure you have installed the Flutter and Dart extensions in VS Code for a better development experience.


4. Configure Utils

Open the mobile/lib/utils directory in the mobile directory and configure the following variables in the utils.dart file:


Replace the following placeholders with your actual values:

  • webApp: URL of the hosted / local web app (e.g., "http://localhost:3000").
  • serverUrl: URL of the hosted / local backend (Node.js) app (e.g., "http://localhost:8800").
  • uploadthingSecret: UploadThing secret key for uploading files to the cloud storage service. Copy UPLOADTHING_SECRET from Legacy tab in UploadThing.
  • razorpayKey: Razorpay key ID for integrating Razorpay payments.
  • currencySymbol: Currency symbol for payment transactions (e.g., "₹").
  • countryCode: Country code (e.g., "IN").
  • currency: Currency for payment transactions (e.g., "usd", "inr").
  • appName: Name of the application (e.g., "EvaluateAI").
